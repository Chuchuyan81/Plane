# SKY ACE 3D: Кампания «Вторжение»
## Полный дизайн-документ кампании из 15 миссий

---

## 📖 1. Лор и Сеттинг

### 1.1 История
**Угроза:** «Пустотный Легион» (Void Legion) — инопланетная раса дронов, стремящаяся захватить атмосферу Земли для терраформирования.

**Протагонист:** Пилот элитного звена «Небесный Туз» (Sky Ace). Вы управляете экспериментальным истребителем-перехватчиком, единственным, способным достигать высот вторжения.

**Контекст миссий:** Земля разделена на сектора обороны. Каждый сектор защищается мощным флагманом Легиона (Боссом).

**Цель:** Прорваться через сектор, уничтожить флагмана и освободить небо. После победы самолёт возвращается в ангар для ремонта и модернизации перед следующим сектором.

### 1.2 Нарративное обоснование механик

| Механика (Code/Design) | Лор-обоснование (Story) |
|------------------------|------------------------|
| **Бесконечный полёт** | Самолёт летит сквозь непрерывный поток вражеских зон перехвата |
| **Чек-поинты** | Сигналы дружественных спутников связи, позволяющие восстановить системы |
| **Бонусы (Power-ups)** | Сброшенные грузы с базы сопротивления или выпавшие из врагов энергоячейки |
| **Рывок (Dash)** | Использование аварийного ускорителя двигателей (имеет перезарядку) |
| **Возрождение (Respawn)** | Системы автоматического восстановления щита и катапультирования |
| **Смена дня/ночи** | Полёт занимает время, самолёт пересекает часовые пояса и зоны боевых действий |
| **Босс** | Командный центр вторжения в конкретном секторе |

## 🎮 2. Структура игрового цикла (Game Loop)

### 2.1 Основные состояния игры
Адаптируем механику «бесконечного полёта» под структуру «Миссия → Босс → Ангар»:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│    MENU     │ ──> │   HANGAR    │ ──> │ MISSION_SELECT │
└─────────────┘     └─────────────┘     └─────────────┘
                           ↑                    │
                           │                    ↓
                    ┌─────────────┐     ┌─────────────┐
                    │ GAMEOVER    │ <── │   PLAYING   │
                    └─────────────┘     └─────────────┘
                           │                    │
                           ↓                    ↓
                    ┌─────────────┐     ┌─────────────┐
                    │   HANGAR    │ <── │BOSS_FIGHT   │
                    └─────────────┘     └─────────────┘
                                               │
                                               ↓
                                        ┌─────────────┐
                                        │MISSION_COMPLETE│
                                        └─────────────┘
```

### 2.2 Этапы цикла

| Этап | Описание | Сохранение |
|------|----------|------------|
| **1. Ангар (Hub)** | Игрок тратит **Кредиты** на улучшения. Выбор следующей миссии. | Прогресс кампании, кредиты, улучшения |
| **2. Запуск Миссии** | Выбор сложности (Easy/Medium/Hard). Загрузка уровня. | Настройки сессии в памяти |
| **3. Полёт (Геймплей)** | Уничтожение дронов (Очки + Кредиты). Сбор бонусов. Чек-поинты. | Чек-поинты по дистанции |
| **4. Битва с Боссом** | При достижении порога очков появляется Босс. Условие победы = завершение миссии. | Чек-поинты `PRE_BOSS`, `MID_BOSS` |
| **5. Награда и Возврат** | **Победа:** Бонус Кредитов, открытие нового региона. **Поражение:** Возврат в Ангар, Кредиты сохраняются. | `MISSION_COMPLETE` → LocalStorage |

## 💰 3. Экономика и Прогрессия

### 3.1 Система валют

| Валюта | Назначение | Сохранение | Источник |
|--------|------------|------------|----------|
| **Очки (Score)** | Рейтинг пилота, таблица лидеров, вызов босса | Сбрасывается после смерти | Уничтожение врагов, комбо |
| **Кредиты (Credits)** | Глобальные улучшения, скины | Сохраняется навсегда | Враги, боссы, достижения, миссии |

### 3.2 Дерево улучшений (за Кредиты)

Используем «Дерево улучшений» из `Концепция игры Самолет.txt`, привязывая его к лору:

| Ветка | Базовая стоимость | Множитель | Макс. уровень | Эффект на уровень |
|-------|-------------------|-----------|---------------|-------------------|
| **Корпус (Hull)** | 100 | 1.5× | 5 | +20 HP к максимуму (старт: 100 HP) |
| **Орудия (Weapons)** | 150 | 1.6× | 5 | +5 к урону, визуальная смена эффектов |
| **Двигатель (Engine)** | 120 | 1.5× | 5 | -0.5с к перезарядке рывка (старт: 5с) |
| **Системы (Systems)** | 100 | 1.4× | 5 | +0.3 к радиусу подбора бонусов |

**Применение улучшений:**
```javascript
// Player.js
applyUpgrades: (upgrades) => {
    this.maxHP = 100 + upgrades.hull * 20;
    this.currentHP = this.maxHP;
    this.damage = 10 + upgrades.weapons * 5;
    this.dashCooldown = 5 - upgrades.engine * 0.5;
    this.pickupRadius = 1 + upgrades.systems * 0.3;
};
```

### 3.3 Скины и коллекционирование
Открываются за достижения (например, «Уничтожить 10 Боссов»). Меняют цвет самолёта и следы двигателей.

| Скин | Требование | Эффект |
|------|------------|--------|
| **Ночной Ястреб** | Пройти Миссию 3 | Тёмная окраска, синие следы |
| **Космический Рейнджер** | Пройти Миссию 7 | Серебристый, белые следы |
| **Золотой Туз** | Пройти Миссию 12 | Золотая окраска, огненные следы |

## 🎨 5. Визуальная эволюция (Миссии)

Используя возможности `Three.js` и описанную смену окружения (`GAME_DESCRIPTION.md`), каждая миссия выглядит уникально:

### Конфигурация окружения
```javascript
// src/config/MissionConfig.js
export const EnvironmentConfig = {
    DAY: { skyColor: 0x87CEEB, cloudDensity: 0.7, visibility: 1.0, groundType: 'terrain' },
    NIGHT: { skyColor: 0x0a0a2a, cloudDensity: 0.5, visibility: 0.8, groundType: 'city_lights' },
    STORM: { skyColor: 0x2a2a3a, cloudDensity: 0.9, visibility: 0.6, groundType: 'ocean', weather: 'rain' },
    STRATOSPHERE: { skyColor: 0x1a2a4a, cloudDensity: 0.3, visibility: 0.9, groundType: 'clouds_below' },
    SPACE: { skyColor: 0x000000, cloudDensity: 0.0, visibility: 1.0, groundType: 'stars' },
    ASTEROID: { skyColor: 0x1a1a2a, cloudDensity: 0.1, visibility: 0.8, groundType: 'asteroids' },
    VOID_CORE: { skyColor: 0x1a0a2a, cloudDensity: 0.3, visibility: 0.7, groundType: 'organic', particles: 'purple' }
};
```

### Применение в Physics.js
```javascript
// src/core/Physics.js
setMissionEnvironment: (environmentType) => {
    const envConfig = EnvironmentConfig[environmentType];
    this.scene.background = new THREE.Color(envConfig.skyColor);
    this.cloudDensity = envConfig.cloudDensity;
    this.visibility = envConfig.visibility;
    this.updateGroundMarkers(envConfig);
    
    // Weather effects
    if (envConfig.weather === 'rain') {
        this.enableRainParticles();
    }
    if (envConfig.particles === 'purple') {
        this.enableVoidParticles();
    }
};
```

### 6.2 Система сохранения кампании
```javascript
// src/utils/storage.js
export const CampaignStorage = {
    saveProgress: (missionId, stars, credits) => {
        const data = JSON.parse(localStorage.getItem('skyace_campaign') || '{}');
        data.completedMissions = data.completedMissions || [];
        if (!data.completedMissions.includes(missionId)) {
            data.completedMissions.push(missionId);
        }
        data.totalCredits = (data.totalCredits || 0) + credits;
        data.currentMission = Math.max(data.currentMission || 1, missionId + 1);
        data.upgrades = data.upgrades || { hull: 0, weapons: 0, engine: 0, systems: 0 };
        localStorage.setItem('skyace_campaign', JSON.stringify(data));
    },
    
    getProgress: () => {
        return JSON.parse(localStorage.getItem('skyace_campaign') || 
            '{"currentMission":1,"completedMissions":[],"totalCredits":0,"upgrades":{"hull":0,"weapons":0,"engine":0,"systems":0}}');
    }
};
```

### 6.3 BossManager для миссий
```javascript
// src/managers/BossManager.js
class BossManager {
    constructor(missionId) {
        this.missionId = missionId;
        this.missionConfig = MissionConfig[missionId - 1];
        this.bossHP = this.calculateBossHP();
    }
    
    calculateBossHP() {
        const baseHP = 50;
        const missionMultiplier = this.missionConfig.bossHPMultiplier;
        const difficultyMultiplier = this.getDifficultyMultiplier(); // Easy: 1.0, Medium: 1.5, Hard: 2.2
        return baseHP * missionMultiplier * difficultyMultiplier;
    }
    
    onBossDefeated() {
        // Вместо продолжения игры - завершение миссии
        GameState.transitionTo('MISSION_COMPLETE');
        CampaignStorage.saveProgress(
            this.missionId, 
            this.calculateStars(), 
            this.missionConfig.creditReward
        );
    }
}
```

### 6.4 Интеграция с существующей системой чек-поинтов
Из `GAME_DESCRIPTION.md` сохраняем логику чек-поинтов, адаптируя под миссии:

| Тип чек-поинта | Когда создаётся | Что сохраняется |
|----------------|-----------------|-----------------|
| `STANDARD` | По интервалам дистанции | Очки, комбо, бонусы, HP |
| `PRE_BOSS` | Перед появлением босса | Состояние перед боем |
| `MID_BOSS` | На Easy при 50% HP босса | Прогресс боя с боссом |
| `POST_BOSS` | После победы над боссом | Для перехода в `MISSION_COMPLETE` |

**Блокировка во время боя:** Дистанция по `Physics` заморожена; обычные чек-поинты не создаются во время `BOSS_FIGHT`.

## 📊 7. Баланс экономики

### 7.1 Формулы расчёта

```
Кредиты за миссию = BaseReward × (1 + 0.1 × Stars)
Стоимость улучшения = BaseCost × (Multiplier ^ Level)
Требуемое количество прохождений = Σ(Cost) / AverageReward
```

### 7.2 Балансовые тесты

| Сценарий | Ожидаемый результат |
|----------|---------------------|
| Игрок проходит Миссию 1 без улучшений | ✅ Успех (базовый HP достаточен) |
| После 3 поражений на Миссии 2 | ✅ Хватает кредитов на 1 улучшение |
| Миссия 15 без максимальных улучшений | ❌ Непроходимо (требуется макс. прокачка) |
| Фарм кредитов на ранних миссиях | ✅ Не ломает прогресс (ограничение по дневному лимиту) |

### 7.3 Кривая сложности

```
Миссия 1-5:   [====] Обучение + базовые механики
Миссия 6-10:  [========] Усложнение + новые враги
Миссия 11-15: [============] Хардкор + требовательность к улучшениям
```
